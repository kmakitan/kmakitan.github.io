Research and Testing Behind the Portfolio

User Research - View the user research done for this portfolio

User Interviews
User interviews allow the researcher to get qualitative information from existing or potential users. Interviews help generate personas that accurately reflect the users. User interviews helped me shape my website to my target audience.

Competitor Analysis
Competitor analysis helps the researcher understand the strengths and weaknesses of other comparable products in the market. The analysis provides insight to potential opportunities to take advantage of and threats that need to be accounted for. Doing a competitor analysis allowed me to see what standards there are for a portfolio and the empty holes I could capitalize on to make mine shine over others.

Feature Value Matrix
A feature value matrix calculates the overall importance of each feature by evaluating it at different values categories. This matrix helped me filter out exactly which features users would like to see on my website.

Feature Prioritization
Using results from the feature value matrix, a feature prioritization graph visualizes the order of importance of each feature. This helps create a development timeline to create elements of a product that are most important first. Prioritizing helped me figure out what was most important and least important to include on my website.

User Testing - View the user testing done for this portfolio

User Stories
User stories are a tool to capture descriptions of a software feature from an end-user perspective. They help identify if the user is able to achieve their goals. Filling out this table helped me understand my users and see if they can accomplish what the goals they have when viewing my website.

Cognitive Walkthrough
A cognitive walkthrough is a usability evaluation method in which evaluators work through a series of tasks and ask a set of questions from the perspective of the user which help the researcher evaluate the learnability of the software for new and infrequent users. Doing a cognitive walkthrough for my website showed me where I needed to improve, such as replacing paragraphs with bullet lists, and helped me redesign so that new users have a better first time experience when visiting my website.

User Test
User testing is a technique used to evaluate a product by testing it on users. A prototype is typically used for testing. Testing involves systematic observation and determines how well real users can use the product. User testing for my website validated the overall structure and design of my portfolio.

Summary of Findings

Conducting user research helped me to develop a portfolio that was catered and optimal to my users. The user interviews defined my target audience and helped me understand what each different group would like to see on a portfolio. The competitor analysis opened my eyes to what kinds of portfolios are out there and the standards that are set. The feature value matrix and feature prioritization allowed me to schedule my development so that I could finish the most important parts first, and the less important sections last.
